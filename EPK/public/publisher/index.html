Start with content, check the public pages, then publish.

Publish safety

Validate, back up, then publish.

  1. Run Check data from the top bar.
  2. Use Backup JSON before live publishing.
  3. Paste a GitHub token only for this session.
  4. Publish live data or create an immutable snapshot.

Security note: this is not a password gate. Protect /publisher/* with Cloudflare Access or equivalent before treating it as private.

Preflight

Before pressing publish

1. Check dataUse the top-bar check and resolve any missing required fields.

2. Backup JSONDownload a copy before changing the live data file.

3. Preview routesOpen the main audience pages and confirm the right content appears.

Publish live EPK data

Commits the current editor data to EPK/public/data/epk.json. Cloudflare then redeploys the live public pages.

Publishing rules

  • Public pages stay clean and do not show the old mode toolbar.
  • This publisher route must be protected with Cloudflare Access if hosted.
  • No token is stored; paste it only when publishing.
  • Snapshots are immutable at /published/<id>/.
